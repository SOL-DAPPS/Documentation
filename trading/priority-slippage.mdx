## Priority Fee

Priority fee determines how quickly your transaction gets processed by validators. Higher fees mean faster execution.

| Mode | Cost | Best for |
|------|------|----------|
| **FAST** | 0.0002 SOL | Most use cases. Already faster than other platforms |
| **ULTRA** | 0.002 SOL | Sniping, arbitrage, or congested network |
| **CUSTOM** | You decide | Full control. Set `mode: "CUSTOM"` and your value in `customValue` |

<Check>
**FAST is the default.** Already faster than most trading platforms.
</Check>

---

## When to Use ULTRA

ULTRA mode is recommended when:

- **Sniping tokens**: You need to be among the first buyers
- **Arbitrage**: Speed is critical for profit
- **Network congestion**: Solana is overloaded and transactions are slow

<Tip>
Only switch to ULTRA when milliseconds matter.
</Tip>

---

## Configuration

<ParamField body="priorityFee.mode" type="string" default="FAST">
  Transaction priority level. Options:
  
  - `FAST`: 0.0002 SOL per transaction (default)
  - `ULTRA`: 0.002 SOL per transaction
  - `CUSTOM`: Use the value specified in `customValue`
</ParamField>

<ParamField body="priorityFee.customValue" type="number | null" default="null">
  Custom priority fee in SOL. Only used when `mode` is set to `CUSTOM`.
  
  Minimum recommended: **0.0002 SOL**
</ParamField>

### Examples

<Tabs>
  <Tab title="FAST (default)">
    ```json
    {
      "priorityFee": {
        "mode": "FAST",
        "customValue": null
      }
    }
    ```
  </Tab>
  
  <Tab title="ULTRA">
    ```json
    {
      "priorityFee": {
        "mode": "ULTRA",
        "customValue": null
      }
    }
    ```
  </Tab>
  
  <Tab title="Custom">
    Set your own priority fee value.
    
    ```json
    {
      "priorityFee": {
        "mode": "CUSTOM",
        "customValue": 0.0005
      }
    }
    ```
    
    <Note>
    Minimum recommended: 0.0002 SOL.
    </Note>
  </Tab>
</Tabs>

## Slippage

Slippage controls how much price movement you accept during a swap. A higher slippage tolerance means your transaction is more likely to succeed, but you may receive fewer tokens.

### Modes

You can specify slippage in two formats:

| Type | Description | Example |
|------|-------------|---------|
| **PERCENT** | Percentage of price movement | 10 = 10% |
| **BPS** | Basis points (1/100 of a percent) | 1000 = 10% |

<Tip>
Use **PERCENT** for simplicity. Use **BPS** for precise control (1 bps = 0.01%).
</Tip>

### Parameters

<ParamField body="slippage.type" type="string" default="PERCENT">
  Slippage unit. Options:
  
  - `PERCENT`: Value represents a percentage (e.g., 10 = 10%)
  - `BPS`: Value represents basis points (e.g., 1000 = 10%)
</ParamField>

<ParamField body="slippage.value" type="number" default="10">
  Slippage tolerance value. Interpreted based on the type.
</ParamField>

### Examples

<Tabs>
  <Tab title="Percentage">
    Accept up to 10% price movement.
    
    ```json
    {
      "slippage": {
        "type": "PERCENT",
        "value": 10
      }
    }
    ```
  </Tab>
  
  <Tab title="Basis Points">
    Accept up to 10% price movement (1000 bps).
    
    ```json
    {
      "slippage": {
        "type": "BPS",
        "value": 1000
      }
    }
    ```
  </Tab>
</Tabs>

<Warning>
Setting slippage too low may cause transactions to fail. Setting it too high may result in fewer tokens received.
</Warning>

## Full Example

```json
{
  "privateKeys": ["WALLET_PRIVATE_KEY"],
  "tokenAddress": "TOKEN_ADDRESS",
  "amount": {
    "type": "SOL",
    "value": 0.5
  },
  "slippage": {
    "type": "PERCENT",
    "value": 10
  },
  "priorityFee": {
    "mode": "FAST",
    "customValue": null
  }
}
```
